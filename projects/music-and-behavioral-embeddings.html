The Cultural Misery Index

Can a playlist tell us how a nation is feeling? That is the animating question behind this project. I built a full multimodal pipeline that fuses three independent data streams, Spotify streaming charts, Genius lyric semantics, and FRED macroeconomic data, into a single analytical engine I call the Cultural Misery Index. The goal: to detect whether American music taste shifts in a statistically measurable way when economic conditions deteriorate.

The project is deliberately not about genres. Instead, I cluster songs purely on their auditory vibe and lyrical themes, then track how those abstract clusters rise and fall alongside Okun's Misery Index. Do songs about anxiety and introspection gain chart share when unemployment spikes? Does party music retreat when inflation bites? By embedding lyrics into dense semantic vectors and fusing them with audio features, I let the data answer those questions without imposing genre labels from the outside.

The Architecture at a Glance

Phase What It Does Key Tools
1. Ingestion Load Spotify charts, fetch lyrics, pull economic series Kaggle CSV, Genius API, FRED API
2. Feature Fusion Embed lyrics (768D), compress to 10D, join with 7 audio features Gemini embedding-001, UMAP, StandardScaler
3. Temporal Clustering Monthly K-Means with stable cluster identities across time scikit-learn KMeans, Hungarian Algorithm
4. Econometric Analysis Measure vibe-to-misery correlation over time Rolling Pearson/Spearman, Granger causality (statsmodels)
5. Narrative Engine AI-generated cultural digest for each cluster Gemini 2.5 Flash, cost-estimated caching layer

1. Dataset Fusion: Three Signals, One Story

The first challenge in this project is that the interesting signal lives at the intersection of three data sources that were never designed to talk to each other. Joining them on a shared temporal axis is the foundational engineering step.

1.1 Spotify Chart Data (Kaggle)

Spotify restricted access to their audio features API for new applications in November 2024. To work around this, I used a pre-compiled Kaggle dataset built before the lockdown. The dataset contains weekly US Top 200 chart entries from 2018 through 2024, with a rich set of pre-extracted audio features for each track.

The audio features I use throughout are: danceability, energy, valence, acousticness, tempo, speechiness, and instrumentalness. Two additional engineered features, harmonic entropy and timbral variance, round out the feature set — and both are derived for free using the iTunes Search API.

The iTunes API is a completely free, no-credential endpoint that returns detailed audio metadata for most commercially released tracks. Given a song title and artist name, it returns a rich JSON payload including preview audio URL, genre, release year, and advisory ratings. From the 30-second preview clip alone, we compute two bespoke acoustic descriptors: harmonic entropy (a measure of tonal unpredictability — how much the chord structure resists expectation) and timbral variance (the spread of spectral color across a track's texture, distinguishing dense multi-instrument productions from sparse, clean arrangements). These two features are derived entirely from freely available audio using librosa, requiring zero Spotify credentials and no paid API access.

from cmi.ingest.spotify_charts import load_charts, get_unique_tracks

charts_df = load_charts(CHART_CSV, region="US", date_start="2018-01-01", date_end="2024-12-31")
unique_tracks = get_unique_tracks(charts_df)

print(f"Chart entries: {charts_df.shape}")
print(f"Unique tracks: {len(unique_tracks)}")

1.2 Okun's Misery Index (FRED)

Arthur Okun's Misery Index is a deliberately simple macroeconomic gauge: the sum of the unemployment rate and the year-over-year inflation rate. When it is high, people are hurting economically. When it is low, the economy feels healthy. I pull both series directly from the Federal Reserve Economic Data (FRED) API and construct the index monthly.

from cmi.ingest.fred_macro import fetch_misery_index

# FRED series: UNRATE (unemployment) + CPIAUCSL (CPI → YoY inflation)
misery_df = fetch_misery_index()
# misery_df has columns: date, unemployment_rate, yoy_inflation, misery_index

1.3 Genius Lyrics

The Genius API is rate-limited, so fetching full lyric sheets for 2,000+ unique tracks is a multi-hour operation on first run. I built an incremental caching system that writes results to disk after each successful fetch. Subsequent runs skip already-cached tracks and only query the API for genuinely new entries. This pattern is critical for a dataset that grows over time.

from cmi.ingest.genius_lyrics import fetch_lyrics_batch

lyrics_df_path = DATA_PROCESSED / "tracks_with_lyrics.csv"

if lyrics_df_path.exists():
    existing_df = pd.read_csv(lyrics_df_path)
    # Only fetch missing tracks — never re-query the API for cached entries
    missing_mask = ~unique_tracks["track_id"].isin(existing_df["track_id"])
    missing_tracks = unique_tracks[missing_mask].copy()
    if len(missing_tracks) > 0:
        new_lyrics_df = fetch_lyrics_batch(missing_tracks)
        combined_df = pd.concat([existing_df, new_lyrics_df], ignore_index=True)
    else:
        combined_df = existing_df
else:
    combined_df = fetch_lyrics_batch(unique_tracks)
Why lyrics matter as much as audio Consider the canonical counterexample: Outkast's Hey Ya! sounds euphoric by every audio metric (high danceability, high energy, high valence) but its lyrics are about a failing relationship. Audio features capture how a song sounds; lyric embeddings capture what it actually means. Fusing both gives us a richer signal than either alone.

2. Feature Engineering — Lyric Embeddings and Multimodal Fusion

This is the technical heart of the project. A raw lyric sheet is just text; it cannot be directly combined with numerical audio features. The solution is to transform each song's lyrics into a dense vector representation using a large language model, then compress that representation down to a dimensionality that is compatible with the audio feature space.

2.1 Lyric Embedding via Gemini

I embed each track's full lyric sheet using Google's gemini-embedding-001 model, which produces a 768-dimensional vector. This vector encodes the full semantic content of the lyrics: themes of isolation, celebration, grief, desire, aggression, and longing all occupy distinct regions of this high-dimensional space.

To avoid re-querying the API on every notebook run, I build a persistent embedding map keyed by track ID. New tracks are embedded and merged into the cache; previously-embedded tracks are loaded directly from disk.

from cmi.features.embeddings import embed_lyrics

# Build a map of track_id -> 768D embedding vector, loading from cache where possible
embedding_map = {}

if embeddings_cache_path.exists():
    old_embeddings = np.load(embeddings_cache_path)
    # Align saved embeddings to the tracks_with_lyrics.csv row order (prefix alignment)
    for idx in range(len(old_embeddings)):
        row = old_tracks_with_text.iloc[idx]
        embedding_map[row["track_id"]] = old_embeddings[idx]

# Embed only the tracks not already in the cache
tracks_to_embed = [lyrics_list[i] for i, tid in enumerate(track_ids) if tid not in embedding_map]
if tracks_to_embed:
    new_embeddings = embed_lyrics(tracks_to_embed)  # → calls gemini-embedding-001

lyric_embeddings = np.vstack(current_embeddings)  # shape: (n_tracks, 768)

2.2 Dimensionality Reduction with UMAP

A 768-dimensional lyric vector would completely overwhelm 7 audio features in any Euclidean distance calculation. The lyric signal would dominate the clustering entirely, making audio features irrelevant. To prevent this, I use UMAP (Uniform Manifold Approximation and Projection) to compress the lyric embeddings from 768 dimensions down to 10, preserving the macro-level semantic structure (the distinction between isolation themes, party themes, and heartbreak themes) while discarding fine-grained nuances that are not relevant at the clustering level.

The Christmas Outliers Holiday music is a recurring nuisance in any chart-based dataset. Because streaming platforms recycle the same catalog of Christmas songs every November and December, these tracks land on the Top 200 year after year — but their lyrical world is entirely alien to the rest of the corpus. UMAP has no trouble identifying them: they cluster tightly in isolation, sequestered from the main body of chart music. In practice, this is a useful sanity check. If UMAP had not separated them, it would be a sign the embeddings were not capturing lyrical semantics meaningfully. 
from cmi.features.fusion import reduce_embeddings, fuse_features

# Compress 768D lyrics → 10D using UMAP
reduced_lyrics = reduce_embeddings(lyric_embeddings, n_components=10)

# Extract and scale the 7 audio features
audio_matrix = extract_audio_matrix(tracks_with_text)

# Concatenate and StandardScale: (n_tracks, 7 audio + 10 lyric = 17 total)
fused_matrix, fused_scaler = fuse_features(audio_matrix, reduced_lyrics)
print(f"Fused matrix: {fused_matrix.shape}")  # → (n_tracks, 17)
The Dimensionality Balance Problem Choosing 10 lyric UMAP components is a deliberate calibration. Too few (e.g., 2D) and you lose the granularity that separates clusters of similar sentiment but different themes, such as party-optimism versus romantic-optimism. Too many (e.g., 50D) and the lyric signal re-dominates the 7-dimensional audio space. At 10D, the ratio is roughly 10:7, giving both modalities meaningful weight in the final clustering.

3. Temporal Clustering — Tracking Vibe Archetypes Over Time

Standard clustering approaches would take all songs across all years and produce a static snapshot. That approach hides temporal dynamics entirely. Instead, I cluster within monthly time windows and use the Hungarian Algorithm to maintain consistent cluster identities as time moves forward.

3.1 Selecting Optimal K — a Semi-Automated Approach

Choosing the number of clusters is notoriously the most subjective step in any K-Means workflow. The classic technique is the "elbow method": plot distortion as a function of k and look for the bend in the curve where adding more clusters stops paying off. Sounds scientific. In practice, data scientists stare at the plot and quietly disagree about whether that bend is an elbow, a knee, a gentle shoulder, or just wishful thinking on a Tuesday afternoon.

To take some of the guesswork out of it, this pipeline uses a two-step, semi-automated selection approach. Step one: Yellowbrick's KElbowVisualizer runs K-Means across a range of k values (4 to 30) and algorithmically detects the sharpest inflection point using a kneedle-style angle computation. This gives us a data-driven candidate rather than a hand-waved one. Step two: a secondary local search runs silhouette scoring over a narrow band around that candidate (±2 values), selecting the final k that maximizes inter-cluster separation while keeping the clusters interpretable. Two passes, two different signals, one number you can actually defend.

from yellowbrick.cluster import KElbowVisualizer
from sklearn.cluster import KMeans
from cmi.clustering.diagnostics import find_optimal_k

model = KMeans(random_state=42, n_init=10)
visualizer = KElbowVisualizer(model, k=(4, 30), force_model=True)
visualizer.fit(fused_matrix)
visualizer.show()

# Refine around the elbow with silhouette scores
optimal_k = find_optimal_k(fused_matrix, k_range=(4, 30))
K_CLUSTERS = optimal_k  # → 9

3.2 The Label-Switching Problem and Hungarian Alignment

K-Means assigns arbitrary integer labels to clusters at each run. If I cluster January's songs and get "Cluster 2 = introspective slow songs," then cluster February's songs and get "Cluster 5 = introspective slow songs," I cannot track the cluster across time — the label has changed but the content has not. This is the label-switching problem, and it is the central engineering challenge in temporal clustering.

The Hungarian Algorithm solves this by treating the problem as a minimum-cost assignment problem. Given the cluster centroids from month t-1 and the new centroids from month t, it finds the optimal 1-to-1 mapping that minimizes the total Euclidean distance between matched centroid pairs. This ensures that "Cluster 2" at month t is always the closest descendant of "Cluster 2" at month t-1, giving us a coherent identity track over time.

from scipy.optimize import linear_sum_assignment

def align_labels(prev_centroids, new_centroids, new_labels):
    """Hungarian algorithm for cluster label alignment across time windows."""
    cost_matrix = cdist(prev_centroids, new_centroids)
    row_ind, col_ind = linear_sum_assignment(cost_matrix)
    # Build permutation: new cluster col_ind[i] → old cluster row_ind[i]
    remap = {col_ind[i]: row_ind[i] for i in range(len(row_ind))}
    aligned = np.vectorize(remap.get)(new_labels)
    return aligned

3.3 Nested Sub-Clustering

Nine top-level clusters capture broad vibe archetypes, but they can mask important sub-structure. For example, one cluster might contain both pure party anthems and ambiguously melancholic dance tracks that superficially share high danceability but carry very different lyrical weight. I apply a second round of K-Means within each parent cluster, splitting each into 2 sub-clusters (M=2), yielding 18 granular sub-profiles total.

The compound sub-cluster ID encodes both levels: sub-cluster 1.0 means parent cluster 1, sub-cluster 0; sub-cluster 1.1 means parent cluster 1, sub-cluster 1. The integer encoding is simply parent_id * 10 + sub_id.

from cmi.clustering.temporal import sub_cluster_parent_clusters

# For each parent cluster, run KMeans(k=2) on the fused vectors of its members
sub_labels = sub_cluster_parent_clusters(fused_matrix, timeline.labels, n_sub_clusters=2)

# Compound encoding: parent_id * 10 + sub_id
tracks_with_text["sub_cluster"] = sub_labels

# Example decoded output:
# Parent Cluster 1 -> Sub-Cluster 0: 228 tracks  (Anxious Era Anthems)
# Parent Cluster 1 -> Sub-Cluster 1: 305 tracks  (No Holds Barred Beats)

4. Econometric Analysis — Does Economic Pain Predict Music Shifts?

With stable, temporally-consistent clusters in hand, I can now measure their relationship to the economy. I compute volumetric shares — the percentage of Top 200 chart entries belonging to each cluster per month — and correlate those time series against the Misery Index.

4.2 Rolling Correlation — The Relationship Evolves

A static correlation averages across all time, hiding the fact that the relationship between music and misery may be stronger in some periods than others. I compute a rolling 6-month Pearson correlation for each cluster to see how the connection shifts across the observation window.

The rolling analysis reveals that the misery-music relationship was most pronounced during 2020-2021, the COVID-19 shock period. During this window, Cluster 5's rolling r with the Misery Index peaked above +0.7 for several consecutive months, while Cluster 6's dropped below -0.6. The relationship weakens during the 2022 inflation period, where the economic driver was price pressure rather than unemployment, suggesting listeners processed inflation-era stress differently from pandemic-era unemployment stress.

4.3 Granger Causality — Does Economic Pain Lead Music Shifts?

Correlation tells us that two variables move together, but it cannot tell us whether economic conditions cause music to shift, or whether both are simply responding to a shared external shock. Granger causality is a statistical test that asks a more pointed question: does knowing the past values of the Misery Index help predict the future values of cluster shares, above and beyond what cluster history alone predicts?

from cmi.correlation.econometrics import granger_causality

# Test whether Misery Index Granger-causes each cluster's volumetric share
# Uses statsmodels grangercausalitytests(), lags 1–3 months
granger_results = granger_causality(cluster_shares, misery_df, max_lag=3)
significant = granger_results[granger_results["p_value"] < 0.05]

# Result: No significant Granger-causal relationships at p < 0.05
The Granger Null Result is Meaningful No cluster passes the Granger causality test at p < 0.05 for any lag (1, 2, or 3 months). This is a genuine and important finding: while music and misery correlate contemporaneously, the economy does not appear to mechanically lead chart behavior by a quarter-month lag. The relationship is likely bidirectional and nearly simultaneous, consistent with music reflecting current collective mood rather than predicting or lagging it. The cultural response to economic shock appears to be immediate, not delayed.

5. Sub-Cluster Narratives — Where the Story Gets Granular

The 9 parent clusters capture broad archetypes. The 18 sub-clusters reveal the real texture of the data. This is where the hierarchical approach pays off: many parent clusters that appear neutral in their Misery Index correlation contain one sub-cluster that is strongly correlated and one that is strongly anti-correlated, pulling the average toward zero.

The Key Insight: Sub-clustering Reveals Masked Signals The parent cluster "Bombastic Beats & Bold Bars" (Cluster 1) has a static Pearson r of only +0.110 with the Misery Index — statistically neutral. But its two sub-clusters tell opposite stories: "No Holds Barred Beats" (r = +0.681) surges during misery, while "Anxious Era Anthems" (r = -0.597) retreats. The parent-level analysis completely cancels these signals out. This is precisely why hierarchical clustering matters: flat K-Means would have obscured this finding entirely.

6. The Gemini Narrative Engine

Numbers describe what happened; narrative explains why it matters. After computing all cluster statistics and correlations, I pass a structured data payload for each cluster to Gemini 2.5 Flash, which generates culturally resonant descriptions that translate abstract centroids into human-readable vibe profiles.

The payload for each cluster includes the audio feature centroid, the top lyric keywords extracted via TF-IDF, the Misery Index correlation value, and the volumetric share delta between the first and second half of the observation window. A cost-estimation and caching layer prevents redundant API calls: if all cluster descriptions are already cached to disk, the engine runs at zero cost.

from cmi.narrative.engine import build_cluster_payload, generate_brief_descriptions

# Build structured payload for each cluster
payloads = []
for cluster_id in range(K_CLUSTERS):
    mask = timeline.labels == cluster_id
    cluster_data = tracks_with_text[mask].copy()
    centroid = cluster_data[AUDIO_FEATURES].mean().values
    corr_val = static_corr[static_corr["cluster"] == f"cluster_{cluster_id}"]["pearson_r"].values[0]

    payload = build_cluster_payload(
        cluster_id=cluster_id,
        cluster_data=cluster_data,
        centroid=centroid,
        temporal_window="2018–2024 (Full Period)",
        misery_correlation=corr_val,
        volume_share_delta=delta_str,
    )
    payloads.append(payload)

# Generate catchy names and cultural descriptions via Gemini 2.5 Flash
brief_descriptions = generate_brief_descriptions(payloads, model="gemini-2.5-flash")
cluster_name_mapping = {d["cluster_id"]: d["name"] for d in brief_descriptions}

Key Findings and Conclusions

1. Music and Misery Correlate, but Do Not Mechanically Cause Each Other

The Granger causality test returned no significant results at any lag from 1 to 3 months. This means the economy does not lead chart behavior by a predictable interval. The relationship is contemporaneous: cultural mood and economic mood appear to shift together, driven by the same underlying events (a pandemic shock, a policy change, an inflation surge) rather than one causing the other.

2. The Clearest Signal Is Escapism Under Pressure

Cluster 5 ("Vibrant Reflection") is the cluster that most consistently expands during economic hardship, with a Spearman r of +0.514 (statistically significant). Its sub-cluster "Groove & Gravitas" (r = +0.383) and the broader "No Holds Barred Beats" (r = +0.681) reinforce this: when times are hard, music that is simultaneously danceable and emotionally complex gains ground. This is consistent with the "escapism through intensity" hypothesis, where difficult times drive demand for music that is simultaneously immersive and emotionally honest.

3. Celebratory Music Retreats in Hard Times

Cluster 6 ("Upbeat Connection Anthems") is the cluster most anti-correlated with misery (Spearman r = -0.442, p = 0.016). It also shows the clearest volumetric decline of -5.4% from the first to second half of the analysis window, which encompasses the COVID and inflation shock years. Feel-good mainstream pop loses chart share precisely when collective economic mood deteriorates.

4. Sub-Clustering Uncovers Masked Signals

The most important methodological finding is the value of hierarchical clustering. Multiple parent clusters appear neutral in their Misery Index correlation, but contain sub-clusters with correlations above +0.5 and below -0.5 that cancel each other out at the aggregate level. Flat single-level clustering would have completely missed these dynamics. The two-level approach reveals that musical archetypes are genuinely heterogeneous: a "trap and hip-hop" cluster contains both anxious introspection and high-energy hedonism, and these sub-vibes behave oppositely under economic stress.

5. The Lyric Embedding Adds Explanatory Power Beyond Audio Alone

The UMAP projection of lyric embeddings (Figure 3) shows that audio valence and lyrical sentiment do not perfectly overlap. High-valence tracks appear throughout the lyric embedding space, not just in the "happy lyrics" corner. This confirms the design hypothesis: if we had clustered on audio features alone, we would have grouped "Hey Ya!" with songs that are actually emotionally simple. The multimodal fusion produces richer, more interpretable archetypes than audio or lyrics alone could generate.

Caveats and Limitations

  • Observation window: The economic overlap of the lyric and Misery Index data is 2019-01 to 2021-05 — a 29-month window that includes the COVID shock but only the beginning of the inflation era. A longer series would provide more statistical power for the Granger tests and correlation estimates.
  • Lyrics coverage: Not all chart tracks have Genius lyrics. Tracks without lyrics are excluded from the embedding and clustering, which may introduce selection bias toward tracks with more extensive lyrical content.
  • The Misery Index is a blunt instrument: Okun's formula treats unemployment and inflation as equally weighted, but their cultural effects may differ substantially. A pandemic spike driven by unemployment may feel very different from an inflation spike, even at the same index value.
  • Correlation does not imply causation. Even with Granger causality ruled out, there may be confounding variables (social media trends, streaming algorithm changes, major cultural events) that simultaneously drive both music taste and economic sentiment.
